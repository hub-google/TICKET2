# Google Flights 爬蟲與資料倉儲系統需求書

## 1. 系統目標
開發一套基於 Python 與 GitHub Actions 的全自動化機票價格追蹤系統。本系統採用「雙庫聯動 (Two-Repo Strategy)」架構，於公開專案 (Public) 利用無限制的 CI/CD 算力執行平行爬蟲任務，並將產出的機票資料與報表自動推送至私密專案 (Private) 進行安全儲存，確保資料隱私與極致的執行效能。

## 2. 抓取目標清單
程式將針對以下航線進行資料抓取：
- **出發地**：台北 (TPE)
- **目的地與預設搜尋條件**：
  - **短程亞洲線 (預設 7 天來回)**：
    東京羽田 (HND)、首爾仁川 (ICN)、新加坡樟宜 (SIN)、吉隆坡 (KUL)、新疆烏魯木齊 (URC)、雲南昆明 (KMG)、越南胡志明市 (SGN)
  - **長程歐美線 (預設 14 天來回)**：
    英國倫敦 (LHR)、法國巴黎 (CDG)、美國奧蘭多 (MCO)、埃及開羅 (CAI)、挪威奧斯陸 (OSL)、義大利羅馬 (FCO)、希臘雅典 (ATH)

## 3. 雙庫聯動與 CI/CD 部署架構 (核心創新)
為了解決單機運算過慢與付費算力的痛點，系統採用完全脫離本地環境的雲端自動化架構：
- **公開運算引擎 (`hub-google/TICKET2`)**：
    - 作為純粹的程式碼載體與 GitHub Actions 執行環境。
    - 利用 GitHub Actions `matrix` 策略，將 14 個城市拆分為 14 台並行虛擬機 (Ubuntu-latest) 同時抓取，將原本需數小時的任務壓縮至 3 分鐘內完成。
    - 各虛擬機在抓取完畢後，將結果暫存為 JSON Artifacts。
- **私密資料金庫 (`hub-google/TICKET2-DATA`)**：
    - 作為儲存 `flights.db` 與 `flights_database.xlsx` 的私密資料倉儲。
    - 公開引擎透過 SSH Deploy Key 建立加密通道，在爬蟲任務結束後，由一台總結機合併所有 JSON Artifacts，並直接將資料與 Excel 報表推送至此私密專案。

## 4. 資料庫設計與報表匯出
- **SQLite 資料庫 (`flights.db`)**：
    - 具備欄位：`origin`, `destination`, `departure_date`, `trip_type`, `scan_date`, `price`, `airline`, `flight_number`, `departure_time`, `arrival_time`, `duration`, `stops`, `booking_url`, `created_at`。
- **Excel 報表 (`flights_database.xlsx`)**：
    - 每日合併資料庫後，透過 `pandas` 與 `openpyxl` 自動讀取資料庫，並匯出成原汁原味的 Excel 檔案供使用者快速查閱分析。

## 5. 爬蟲架構與演算法
- **通訊協定升級**：使用支援 HTTP/2 協議的 `httpx`，以極大化網路傳輸效能與降低連線延遲。
- **視障模式 (aria-label) 精準打擊**：利用 Google Flights 無障礙標籤 (`aria-label`) 提取白話文資訊，透過正則表達式 (Regex) 暴力萃取價格、航空公司與起降時間，避開複雜 DOM 解析。
- **動態頻率增量掃描 (Smart Delta Scanning)**：
    - **未來 1 ~ 30 天**：每天必掃。
    - **未來 31 ~ 180 天**：每 2 天掃描一次。
    - **未來 181 ~ 330 天**：每 5 天掃描一次。
    - 此演算法將每日掃描數量從 9,240 筆壓縮至約 3,780 筆，降低被阻擋風險。
- **同日智能去重與補漏 (Smart Upsert)**：
    - 以 `scan_date` 為基準，對每一筆航班進行深度比對。若該日已掃描過，則不重複新增，僅對價格與詳細資訊進行智慧合併與修正，確保資料庫精簡且沒有冗餘數據。

## 6. 自動化維運
- **無人值守**：設定每日 UTC 18:00 (台灣凌晨 02:00) 準時自動觸發工作流。
- **模組化腳本**：
    - `爬蟲.py`：負責單點城市抓取。
    - `merge_db.py`：負責下載、合併 JSON 暫存檔至 SQLite。
    - `export_excel.py`：負責產出 Excel 報表。
