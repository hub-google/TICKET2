# Google Flights 爬蟲系統需求書

## 1. 系統目標
開發一支基於 Python 的全新 Google Flights 網頁爬蟲程式 (`爬蟲.py`)，自動抓取台北 (TPE) 出發至指定亞洲與歐美城市的航班資訊，並將結果結構化儲存至本地的 SQLite 資料庫中。

## 2. 抓取目標清單
程式將針對以下航線進行資料抓取：
- **出發地**：台北 (TPE)
- **目的地與預設搜尋條件**：
  - **短程亞洲線 (預設 7 天來回)**：
    - 東京羽田 (HND)、首爾仁川 (ICN)、新加坡樟宜 (SIN)、吉隆坡 (KUL)、新疆烏魯木齊 (URC)、雲南昆明 (KMG)、越南胡志明市 (SGN)
  - **長程歐美線 (預設 14 天來回)**：
    - 英國倫敦 (LHR)、法國巴黎 (CDG)、美國奧蘭多 (MCO)、埃及開羅 (CAI)、挪威奧斯陸 (OSL)、義大利羅馬 (FCO)、希臘雅典 (ATH)

## 3. 資料庫設計 (SQLite)
爬取下來的資料將存入本地端資料庫 (`flights.db`)，資料表 (`flights`) 應具備以下欄位：
- `origin` (TEXT): 出發地機場代碼
- `destination` (TEXT): 目的地機場代碼
- `departure_date` (TEXT): 航班起飛日期 (YYYY-MM-DD)
- `trip_type` (TEXT): 搜尋模式 (單程/來回)
- `price` (INTEGER): 機票價格，以 TWD 計價
- `airline` (TEXT): 航空公司名稱
- `flight_number` (TEXT): 航班號碼
- `departure_time` (TEXT): 起飛時間
- `arrival_time` (TEXT): 抵達時間
- `duration` (TEXT): 總飛行時間
- `stops` (TEXT): 轉機資訊 (空字串 `""` 代表直飛，否則寫入如 `"轉機 1 次"` 等字串)
- `booking_url` (TEXT): 快速訂票連結

## 4. 爬蟲架構與技術選型
- **爬蟲引擎**：從 `requests` 全面升級至支援 HTTP/2 協議的多路復用用戶端 `httpx` (`httpx.Client(http2=True)`)，以極大化網路傳輸效能與降低連線延遲。
- **視障模式 (aria-label) 精準打擊**：利用 Google 為了符合無障礙網頁規範所提供的 `aria-label` 屬性。利用 `BeautifulSoup` 抓出隱藏了完整白話文資訊的 `aria-label` 標籤，並透過正則表達式 (Regex) 暴力萃取價格、航空公司、起降時間、轉機等資訊，避開繁雜且變動的 DOM 結構。
- **動態頻率增量掃描 (Smart Delta Scanning)**：考量遠期機票價格變動較低，為保護 IP 與節省運算資源，實作智慧型輪替抓取演算法：
    *   **未來 1 ~ 30 天**：每天必掃。
    *   **未來 31 ~ 180 天**：每 2 天掃描一次。
    *   **未來 181 ~ 330 天**：每 5 天掃描一次。
    *   總請求量藉此從每日 9,240 筆大幅降低約 60% 至 3,780 筆。
- **航班號碼萃取**：利用隱藏的 `itinerary` 屬性（如 `itinerary=TPE-NRT-BR-198-20261002`），以 Regex 精準切分出真實的航班號碼。
- **資料庫整合**：使用 Python 的 `sqlite3` 進行輕量級本地儲存，確保每次啟動都會自動檢查並建立 Table。
- **同日智能去重與補漏 (Smart Upsert / De-duplication)**：為避免同一天多次掃描導致無效資料膨脹，寫入前會以 `scan_date` 為基準比對：
    *   若有班機資料但因錯誤未抓到，價格存為 `NULL`。
    *   若確認當天沒有班機 (無航班資料)，價格存為 `-1`。
    *   若無同航班紀錄，則正常新增。
    *   若原本為 `NULL`，這次確認為無航班 (`-1`)，則更新為 `-1`。
    *   若原本為空值 (`NULL` 或 `-1`)，這次有抓到票價則補漏更新。
    *   若原本已有有效票價，這次未抓到（`NULL` 或是 `-1`）則跳過不覆蓋（保護有效資料）。
    *   若兩次皆有有效票價，則更新為最新價格。
- **爬蟲防護處理**：設定合理的隨機 `User-Agent`，並模擬真實瀏覽器 Headers 以成功取得內容。
- **日誌記錄 (Logging)**：執行過程中需詳細記錄每筆航線的狀態（成功、失敗、找不到資料），並輸出至獨立的 Log 檔案（如 `scraper.log`），以便後續追蹤與錯誤分析。
